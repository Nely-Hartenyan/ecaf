# Добавление участников в проект Vercel

## Как добавить участников в проект на Vercel

### Шаг 1: Откройте настройки проекта

1. Зайдите на [vercel.com](https://vercel.com)
2. Откройте ваш проект `ecaf`
3. Перейдите в **Settings** (Настройки)
4. В левом меню выберите **Team** (Команда) или **Members** (Участники)

### Шаг 2: Добавьте участника

1. Нажмите **Add Member** (Добавить участника) или **Invite** (Пригласить)
2. Введите email адрес участника
3. Выберите роль:
   - **Owner** (Владелец) - полный доступ ко всему
   - **Developer** (Разработчик) - может деплоить, изменять настройки
   - **Viewer** (Наблюдатель) - только просмотр

   **Рекомендуется:** Выберите **Developer** для разработчиков

4. Нажмите **Send Invitation** (Отправить приглашение)

### Шаг 3: Участник принимает приглашение

Участник получит email с приглашением:
1. Нажмет на ссылку в email
2. Войдет в свой аккаунт Vercel (или создаст новый)
3. Приглашение будет принято автоматически

---

## Что смогут делать участники

### С ролью Developer:

✅ **Могут:**
- Видеть проект в своем Dashboard
- Выполнять `vercel link` для подключения
- Получать переменные окружения через `vercel env pull`
- Видеть деплои и логи
- Создавать новые деплои
- Изменять настройки проекта (кроме удаления)

❌ **Не могут:**
- Удалять проект
- Удалять команду
- Изменять биллинг

### С ролью Viewer:

✅ **Могут:**
- Только просматривать проект
- Видеть деплои и логи
- НЕ могут изменять настройки

## Инструкция для нового участника

После того, как вы добавили участника, он должен выполнить:

### 1. Клонировать репозиторий

```bash
git clone git@github.com:Nely-Hartenyan/ecaf.git
cd ecaf
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Установить Vercel CLI

```bash
npm i -g vercel
```

### 4. Войти в Vercel

```bash
vercel login
```

### 5. Подключиться к проекту

```bash
vercel link
```

При выполнении `vercel link`:
- Выберите **Link to existing project**
- Выберите проект `ecaf`
- Выберите scope (ваш аккаунт или команда)

### 6. Получить переменные окружения

```bash
vercel env pull .env.local
```

Это создаст файл `.env.local` с переменными из Vercel.

### 7. Запустить проект

```bash
npm run dev
```

✅ **Готово!** Участник может работать с проектом.

## Альтернатива: Добавить в GitHub команду

Если проект на GitHub, также можно добавить участников туда:

### На GitHub:

1. Откройте репозиторий на GitHub
2. Перейдите в **Settings** → **Collaborators**
3. Нажмите **Add people**
4. Введите username или email
5. Выберите права доступа:
   - **Read** - только чтение
   - **Write** - может коммитить
   - **Admin** - полный доступ

**Рекомендуется:** Дайте права **Write** для разработчиков

## Проверка доступа

Участник может проверить, что у него есть доступ:

```bash
# Проверить подключение к Vercel
vercel whoami

# Посмотреть список проектов
vercel projects

# Посмотреть переменные окружения (должен видеть проект)
vercel env ls
```

---

## Управление доступом

### Изменить роль участника:

1. Vercel Dashboard → Settings → Team/Members
2. Найдите участника
3. Нажмите на его роль
4. Выберите новую роль
5. Сохраните

### Удалить участника:

1. Vercel Dashboard → Settings → Team/Members
2. Найдите участника
3. Нажмите **Remove** (Удалить)
4. Подтвердите

## Важные замечания

### Безопасность:

- ⚠️ Участники с доступом к Vercel могут видеть переменные окружения (включая секреты)
- ⚠️ Убедитесь, что доверяете участникам
- ⚠️ Используйте роли правильно (не давайте Owner всем)

### Переменные окружения:

- Все участники с ролью Developer могут видеть переменные окружения
- Они могут получить их через `vercel env pull`
- Это нормально для команды разработчиков

### Git доступ:

- Доступ к Vercel НЕ дает автоматический доступ к GitHub
- Нужно отдельно добавить участников в GitHub репозиторий
- Или использовать GitHub Teams для управления доступом

## FAQ

**Q: Нужно ли участнику иметь доступ к GitHub?**
A: Да, если он хочет клонировать репозиторий. Добавьте его в GitHub отдельно.

**Q: Может ли участник видеть секреты?**
A: Да, участники с ролью Developer могут видеть переменные окружения через `vercel env pull`. Это нормально для команды.

**Q: Может ли участник удалить проект?**
A: Только если у него роль Owner. Developer не может удалить проект.

**Q: Как отозвать доступ?**
A: Vercel Dashboard → Settings → Team/Members → Remove
