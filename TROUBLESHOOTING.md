# Решение проблем

## Проблема: Новости не отображаются на сайте после создания

### Причины:

1. **Чекбокс "Published" не отмечен**
   - Убедитесь, что при создании новости отмечен чекбокс "Published"
   - Без этого новость не будет видна на публичном сайте

2. **Кеширование на Vercel**
   - Vercel может кешировать страницы
   - После создания новости подождите 10-30 секунд и обновите страницу (Ctrl+F5 или Cmd+Shift+R)

3. **Проблема с revalidatePath**
   - Код уже обновлен для правильной инвалидации кеша
   - Если проблема сохраняется, попробуйте:
     - Очистить кеш браузера
     - Открыть сайт в режиме инкогнито
     - Подождать несколько минут

### Решение:

1. **Проверьте статус новости в админ-панели:**
   - Зайдите в `/admin/news`
   - Убедитесь, что новость имеет статус "Published" (зеленая галочка)

2. **Если новость не опубликована:**
   - Откройте новость для редактирования
   - Отметьте чекбокс "Published"
   - Сохраните

3. **Если новость опубликована, но не видна:**
   - Подождите 30 секунд
   - Обновите страницу новостей (Ctrl+F5 / Cmd+Shift+R)
   - Проверьте, что новость действительно сохранена в базе данных

4. **Проверка через Prisma Studio:**
   ```bash
   vercel link
   vercel env pull .env.local
   npx prisma studio
   ```
   - Откройте модель `News`
   - Проверьте, что `published: true`
   - Проверьте, что `publishedAt` установлен

### Быстрая проверка:

1. Создайте новую новость с чекбоксом "Published" ✅
2. Сохраните
3. Подождите 10-30 секунд
4. Откройте `/news` на сайте
5. Новость должна появиться

### Если ничего не помогает:

1. Проверьте логи на Vercel:
   - Зайдите в Vercel Dashboard
   - Откройте ваш проект
   - Перейдите в "Functions" → "Logs"
   - Проверьте, нет ли ошибок при сохранении

2. Проверьте базу данных напрямую:
   ```bash
   vercel env pull .env.local
   npx prisma studio
   ```

3. Создайте новость заново с простым названием и контентом для теста
