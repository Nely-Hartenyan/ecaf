# Быстрый старт для разработки

## Проблема: "User was denied access on the database"

Эта ошибка означает, что база данных недоступна. Решение:

## Решение 1: Запустить локальную базу данных (РЕКОМЕНДУЕТСЯ)

### Шаг 1: Запустите Docker Desktop
Откройте Docker Desktop на вашем Mac и дождитесь его полной загрузки.

### Шаг 2: Запустите базу данных
```bash
npm run db:up
```

### Шаг 3: Примените миграции
```bash
npm run db:migrate
```

### Шаг 4: Заполните базу данных (опционально)
```bash
npm run db:seed
```

### Шаг 5: Запустите проект
```bash
npm run dev
```

## Решение 2: Использовать базу данных из Vercel

Если у вас уже настроена база данных на Vercel:

1. Получите переменные окружения:
   ```bash
   vercel env pull .env.local
   ```

2. Или вручную отредактируйте `.env.local` и замените `DATABASE_URL` на значение из Vercel.

3. Перезапустите dev сервер:
   ```bash
   npm run dev
   ```

## Проверка подключения

После настройки проверьте подключение:
```bash
npm run db:studio
```

Если Prisma Studio открылся - база данных работает!

## Все команды одной строкой

Для локальной разработки:
```bash
npm run db:up && sleep 3 && npm run db:migrate && npm run db:seed && npm run dev
```
