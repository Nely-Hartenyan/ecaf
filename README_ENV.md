# Работа с переменными окружения

## Файл .env.local от Vercel

Если вы получили файл `.env.local` от Vercel (через `vercel env pull`), это переменные окружения из вашего production проекта.

### Что делать с .env.local:

1. **Сохраните файл в корне проекта** (рядом с `package.json`)
   - Файл уже в `.gitignore`, поэтому он не попадет в Git
   - Это безопасно - секреты не будут закоммичены

2. **Используйте для локальной разработки:**
   ```bash
   # Файл .env.local автоматически загружается Next.js
   npm run dev
   ```

3. **Проверьте содержимое:**
   Убедитесь, что в файле есть:
   - `DATABASE_URL` - для подключения к базе данных
   - `NEXTAUTH_URL` - URL вашего приложения
   - `NEXTAUTH_SECRET` - секретный ключ
   - `BLOB_READ_WRITE_TOKEN` - если используете Blob Storage

### Важно:

- ⚠️ **НЕ коммитьте** `.env.local` в Git (уже в `.gitignore`)
- ✅ Используйте `.env.local` только для локальной разработки
- ✅ Для production все переменные уже настроены на Vercel
- ✅ Если нужно обновить переменные на Vercel, используйте веб-интерфейс или `vercel env add`

### Команды для работы с переменными Vercel:

```bash
# Получить переменные из Vercel (создает .env.local)
vercel env pull .env.local

# Добавить новую переменную
vercel env add DATABASE_URL

# Посмотреть все переменные
vercel env ls
```

### Для локальной разработки с базой данных:

Если вы хотите использовать локальную базу данных для разработки:

1. Создайте файл `.env` (не `.env.local`) на основе `env.example`
2. Настройте локальную PostgreSQL базу данных
3. `.env.local` будет иметь приоритет над `.env`

Или просто используйте `.env.local` с production базой данных для тестирования.
